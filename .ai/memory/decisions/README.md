# Architecture Decision Records

This directory contains Architecture Decision Records (ADRs) documenting significant technical decisions.

## Format

Each ADR follows this template:

```markdown
# ADR-XXX: Title

## Status
Proposed | Accepted | Deprecated | Superseded by ADR-XXX

## Context
What is the issue that we're seeing that is motivating this decision?

## Decision
What is the change that we're proposing and/or doing?

## Consequences
What becomes easier or more difficult to do because of this change?
```

## Creating New ADRs

1. Create a new file: `adr-XXX-short-title.md`
2. Use the template above
3. Add to the index in this README
4. Commit with message: `docs: add ADR-XXX <title>`
